================================================================================
THE SIGNAL — INVENTORY ITEM GUIDE
================================================================================
Last Updated: 2026-03-03
Purpose: Consolidated asset inventory, item recommendations, and level designer
         guidelines for placing and creating items.

Systems Used:
  - Moonville Inventory System Pro (plugin — item storage, UI, crafting, pickups)
  - Kinemation Tactical Shooter Pack (plugin — FPS weapon models + animations)
  - Custom C++ bridge (BP_KinemationEquipAction — connects inventory to weapons)

PART 4: LEVEL DESIGNER GUIDE — PLACING ITEMS IN THE MAP
================================================================================

You are placing items in UE5 editor. You do NOT need to write code.
These instructions use exact editor labels you'll see on screen.

--- PLACING A SIMPLE PICKUP (weapon, ammo, medkit, key) -----------------------

  1. In Content Browser, navigate to:
     InventorySystemPro > Blueprints > Actors > Pickup

  2. Drag "BP_ItemPickup" into the viewport where you want the item.

  3. With the pickup selected, look at the Details panel on the right.

  4. Find the property "Item" (under the Pickup category).
     Click the dropdown and select the item DataAsset you want:
       - DA_Grace_Pistol     (Viper handgun)
       - DA_Medkit            (First aid kit)
       - DA_Ammo_9mm          (Handgun ammo)
       - DA_Ammo_Buckshot     (Shotgun shells)
       - DA_Keycard            (Access card)
       - DA_DoorKeyBlack       (Physical key)
       - etc.

  5. Set "Amount" to how many the player gets (1 for weapons/keys,
     12 for 9mm ammo, 4 for buckshot, etc.)

  6. The pickup will use the item's mesh and appear in the world.
     Player walks up, sees interaction popup, presses E, item goes
     to inventory.

--- PLACING A CONTAINER (chest, crate, supply cabinet) -------------------------

  1. Drag "BP_ItemPickupChest" or "BP_ItemContainer_Horror" into the level.

  2. In Details, find the "Items" array.

  3. Add entries — each entry is an item DA + amount.
     Example supply cabinet:
       Entry 0: DA_Medkit, Amount 1
       Entry 1: DA_Ammo_9mm, Amount 12
       Entry 2: DA_Ammo_Buckshot, Amount 4

  4. Player interacts → container UI opens → player takes what they want.

--- PLACING A LOCKED DOOR ------------------------------------------------------

  1. Drag "BP_InventoryDoor" into the level at a doorway.

  2. In Details, find "Required Item" and set it to the key DA:
       - DA_Keycard            (electronic door)
       - DA_DoorKeyBlack       (mechanical door)
       - DA_MedallionComplete  (puzzle door — needs combined medallion)

  3. Set "Consume Key" to true if the key should be removed from inventory
     after use (one-time keys). Set false for reusable keycards.

  4. Player approaches door → "Locked. Requires [item name]" prompt.
     If player has the item → door opens. If not → stays locked.

--- SETTING UP THE COMBINE PUZZLE (medallion) ----------------------------------

  The medallion combine recipe already exists in Moonville:
  DA_RecipeMedallion (Left + Right → Complete)

  1. Place "BP_ItemPickup" in East wing:
     Item = DA_MedallionLeftPart, Amount = 1

  2. Place "BP_ItemPickup" in West wing:
     Item = DA_MedallionRightPart, Amount = 1

  3. Player picks up both halves.

  4. In inventory, player selects one half → context menu → "Combine."
     If other half is in inventory, they merge into DA_MedallionComplete.

  5. Place "BP_InventoryDoor" at exit:
     Required Item = DA_MedallionComplete, Consume Key = true

  That's the full key-combine-lock loop. No code needed.

--- PLACING AN ITEM TRIGGER (for scripted events) -----------------------------

  1. Drag "BP_HasItemTrigger" into the level as a volume.

  2. Set "Required Item" to the item DA you want to check for.

  3. Wire the trigger's output event to whatever you want to happen
     (open a door, play a sequence, spawn enemies, trigger dialogue).

  4. Player enters the volume WITH the required item → event fires.
     Without the item → nothing happens.

PART 5: LEVEL DESIGNER GUIDE — CREATING NEW ITEMS
================================================================================

When you need an item that doesn't exist yet. No code required for basic items.

--- CREATING A NEW ITEM DATAASSET ----------------------------------------------

  1. In Content Browser, navigate to: Core > Items

  2. Right-click → Miscellaneous → Data Asset

  3. Select parent class: "PDA_Item" (Moonville's base item class)

  4. Name it following convention: DA_[ItemName]
     Examples: DA_LabKeycard, DA_Syringe, DA_Flashlight, DA_SecurityBadge

  5. Open the new DA. Fill in these fields:

     REQUIRED:
       Item Name:        Display name shown in UI ("Lab Keycard")
       Item Description: Tooltip text ("Level 3 access card for the east wing")
       Item Icon:        Texture for inventory grid (drag from Content Browser)
       Item Mesh:        Static mesh shown in world pickup (drag from CB)
       Grid Size:        How many inventory slots it takes (X=1, Y=1 for small)
       Max Stack:        How many can stack (1 for unique keys, 99 for ammo)
       Item Tags:        Categories from DT_InventoryTags (Weapon, Consumable,
                         KeyItem, Ammo, etc.)

     OPTIONAL:
       Item Weight:      For weight-based inventory limits (if enabled)
       Item Rarity:      Common/Uncommon/Rare/Epic/Legendary (affects UI border)
       Item Action:      What happens when used (see below)
       Can Drop:         Whether player can drop this item (false for quest keys)
       Can Examine:      Whether player can 3D-inspect this item
       Examine Mesh:     Detailed mesh shown in inspection viewer

  6. Save. The item now exists and can be placed via BP_ItemPickup.

--- ITEM ACTIONS (what happens when the player "uses" an item) -----------------

  Pre-built action classes (no code needed):

    BP_ConsumeHealthAction   — Restores health. Set "Heal Amount" on the DA.
    BP_EquipItemAction       — Equips to character (armor, accessories).
    BP_ExpandInventoryAction — Adds inventory slots.
    BP_KinemationEquipAction — Equips a Kinemation FPS weapon (our custom bridge).

  To assign: Open the item DA → find "Item Action Class" → select from dropdown.

  For items with NO action (keys, story items, puzzle pieces):
    Leave Item Action Class empty. These items exist in inventory but
    aren't "used" — they're checked by doors/triggers.

--- CREATING A NEW HEALING ITEM ------------------------------------------------

  1. Duplicate DA_Medkit (right-click → Duplicate)
  2. Rename to DA_[YourItem] (e.g., DA_Bandage)
  3. Open it, change:
     - Item Name: "Bandage"
     - Item Description: "Basic wound dressing. Restores a small amount of health."
     - Item Icon: your icon texture
     - Item Mesh: your mesh
     - Heal Amount: 25 (or whatever)
     - Item Action Class: BP_ConsumeHealthAction
  4. Save. Place via BP_ItemPickup.

--- CREATING A NEW KEY ITEM ----------------------------------------------------

  1. Duplicate DA_DoorKeyBlack or DA_Keycard
  2. Rename to DA_[YourKey] (e.g., DA_LabKeycard_East)
  3. Open it, change:
     - Item Name: "East Wing Keycard"
     - Item Description: "Electronic access card. Level 2 clearance."
     - Can Drop: false (player shouldn't be able to discard quest keys)
     - Item Action Class: (leave empty — keys aren't "used," doors check for them)
  4. Save. Place the key via BP_ItemPickup, place the door via BP_InventoryDoor
     with Required Item = your new DA.

--- CREATING A NEW WEAPON ITEM (Kinemation FPS weapon) -------------------------

  This requires a C++ change to support multiple weapon classes.
  Once that's done, the process is:

  1. Duplicate DA_Grace_Pistol
  2. Rename to DA_[WeaponName] (e.g., DA_PoliceShotgun)
  3. Open it, change:
     - Item Name: "Police Shotgun"
     - Item Description: "Herrington 11-87 tactical. Facility security issue."
     - Item Icon: shotgun icon
     - Item Mesh: SM_Shotgun (for world pickup display)
     - Grid Size: X=2, Y=1 (shotgun is bigger than pistol)
     - Item Action Class: BP_KinemationEquipAction
     - (Once C++ supports it) Weapon Class: BP_Herrington_11-87_Police
  4. Save. Place via BP_ItemPickup.

  NOTE: Until the C++ multi-weapon support is built, all weapon items
  will equip the same pistol. This is the first thing to implement.

--- USING ASSETS FROM OUTSIDE MOONVILLE ----------------------------------------

  Any static mesh can be an item. You're not limited to Moonville's meshes.

  To use a TreatmentStation prop as a pickup:
  1. Create a new DA as described above
  2. Set Item Mesh to the TreatmentStation mesh (e.g., SM_ElectricalBox_1)
  3. Create or assign an icon texture (screenshot the mesh, crop to square,
     import as texture — or use a placeholder)
  4. Everything else works the same

  To use a mesh from any Marketplace pack:
  Same process. The inventory system doesn't care where the mesh comes from.
  It just needs: a DataAsset, a mesh, and an icon.

--- ICON CREATION TIPS ---------------------------------------------------------

  Moonville expects square icon textures (recommended 256x256 or 512x512).

  Quick method:
  1. Place the mesh in an empty level
  2. Position camera, take a screenshot (F9 in viewport)
  3. Crop to square in any image editor
  4. Import as texture, set compression to "UserInterface2D"
  5. Assign to your item DA's Icon field

  Professional method:
  Use UE5's Scene Capture 2D to render the mesh to a Render Target,
  then export as texture. Moonville's example items were made this way.
